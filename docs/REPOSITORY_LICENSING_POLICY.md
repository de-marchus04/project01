# Repository Licensing Policy

## Current Position

YogaPlatform is currently treated as a proprietary commercial product.

Default rule:

- All rights are reserved unless a separate written license explicitly grants broader usage rights.

This policy is intended to avoid accidental open-source signaling while the product, brand, content library, and internal methodology are still being formed.

## Why This Position Is Used Now

- The product is intended for future premium commercialization.
- The repository contains business logic, branded product materials, and strategic documentation.
- The project still needs a formal audit of third-party dependencies and asset provenance.
- Public open-source licensing before that audit creates avoidable legal and ownership risk.

## Practical Rules

1. Do not add a permissive open-source license file unless there is an explicit business decision to do so.
2. Keep package metadata aligned with the proprietary position.
3. Do not copy third-party code or content into the repository without documenting source and license.
4. Treat design systems, templates, seeded content, and future AI logic as protected internal assets by default.

## If Licensing Changes Later

If the project later adopts a different distribution model, update all of the following together:

- package.json
- README.md
- repository-level license files
- contributor agreements
- distribution and publishing settings

Partial updates are not acceptable because they create contradictory legal signals.
